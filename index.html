⚙️ Configurações — Banco de dados

🟡
Modo local — dados salvos apenas neste navegador
Configure o Supabase abaixo para compartilhar dados com a equipe em tempo real.
🗄️ Credenciais do Supabase
Encontre em: Settings → General → Project URL
Encontre em: Settings → Chaves da API → "Chave publicável" (começa com sb_publicable_) ou aba Legacy → anon public
💡 Como obter as credenciais
1. Acesse supabase.com → seu projeto
2. No menu lateral clique em Settings → Chaves da API
3. Aba "Chaves de API publicáveis e secretas" → copie a Chave publicável (começa com sb_publicable_)
4. Vá em Settings → General → copie a Project URL
5. Cole os dois campos acima e clique em Salvar e conectar
⚠️ Ainda não criou as tabelas no Supabase? Execute o SQL abaixo no SQL Editor do Supabase antes de conectar:
CREATE TABLE IF NOT EXISTS empreendimentos (id BIGSERIAL PRIMARY KEY, nome TEXT NOT NULL, cnpj TEXT, resp TEXT, email TEXT, tel TEXT, "end" TEXT, mun TEXT, uf TEXT DEFAULT 'PR', cep TEXT, nlic TEXT, tlic TEXT, demis DATE, dvenc DATE, status TEXT DEFAULT 'Ativa', "tipoAgua" TEXT, outorga TEXT, efluente TEXT, "tipoEfluente" TEXT, reuso TEXT, "corpoReceptor" TEXT, created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE IF NOT EXISTS pendencias (id BIGSERIAL PRIMARY KEY, emp TEXT NOT NULL, tipo TEXT, desc TEXT, dabertura DATE, prazo DATE, status TEXT DEFAULT 'Aberto', resp TEXT, obs TEXT, created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE IF NOT EXISTS analises (id BIGSERIAL PRIMARY KEY, emp TEXT NOT NULL, tipo TEXT, params TEXT, period TEXT, ultima DATE, proxima DATE, lab TEXT, laudo TEXT, resultado TEXT DEFAULT 'Aguardando', status TEXT DEFAULT 'Aguardando', created_at TIMESTAMPTZ DEFAULT NOW()); CREATE TABLE IF NOT EXISTS mtrs (id BIGSERIAL PRIMARY KEY, num TEXT, demis DATE, emp TEXT NOT NULL, transp TEXT, dest TEXT, residuo TEXT, ibama TEXT, qtd NUMERIC, drec DATE, status TEXT DEFAULT 'Pendente', created_at TIMESTAMPTZ DEFAULT NOW()); ALTER TABLE empreendimentos ENABLE ROW LEVEL SECURITY; ALTER TABLE pendencias ENABLE ROW LEVEL SECURITY; ALTER TABLE analises ENABLE ROW LEVEL SECURITY; ALTER TABLE mtrs ENABLE ROW LEVEL SECURITY; CREATE POLICY "pub_emp" ON empreendimentos FOR ALL USING (true) WITH CHECK (true); CREATE POLICY "pub_pend" ON pendencias FOR ALL USING (true) WITH CHECK (true); CREATE POLICY "pub_anal" ON analises FOR ALL USING (true) WITH CHECK (true); CREATE POLICY "pub_mtrs" ON mtrs FOR ALL USING (true) WITH CHECK (true);
